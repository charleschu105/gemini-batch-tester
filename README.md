# Gemini 影像辨識批次測試工具

這是一個基於 **CustomTkinter** 打造的現代化 Python 桌面應用程式，專為批次測試 Google Gemini 大模型在物品影像辨識上的準確度而設計。本工具專注於**純本機處理**，可自動比對圖檔的「真實標籤（Ground Truth）」與「模型預測結果（Predicted）」，並將結果輸出為 Excel 友善的 CSV 報表。

---

## 🌟 核心特色

1. **精美現代化介面**：採用 CustomTkinter 打造深色主題 (Dark Mode) UI，具備高對比視覺與圓角設計，並支援高解析度 (DPI) 縮放。
2. **本機安全防卡死**：採用背景執行緒 (Multi-threading) 技術處理 API 請求，進度條流暢更新，視窗絕不凍結，並支援隨時「停止執行」。
3. **設定自動記憶 (`config.ini`)**：在關閉視窗或執行時，會自動將您的 API 金鑰、模型選擇、自訂提示詞與上次選取的資料夾路徑儲存於相鄰的 `config.ini` 中，下次開啟自動回復狀態。
4. **真實標籤自動解析**：
   * 圖檔可命名為：`物品名稱_分類_敘述_編號.ext` (例如 `蘋果_食品_香甜_01.jpg` 或 `牛奶_食品_EX20260811_02.png`)。
   * 程式將自動拆解並提取出其**原物品名稱**、**原分類**、**原敘述**與**原編號**寫入 CSV 中。
5. **過期日期智慧比對**：
   * 若「原敘述」是以 `EX` 開頭（大小寫不限，如 `EX20260811`），系統將自動將其與模型預測的有效日期進行**智慧標準化對比**（消除 `-`、`/`、`年`、`月`、`日` 等符號差異）。
   * 一致時輸出 `OK`，不一致時輸出 `FAIL`；若原敘述非以 `EX` 開頭，則代表無日期標籤，輸出 `NA`。
6. **實驗名稱提取**：自動將所選取圖檔的「資料夾名稱」視為「本次實驗名稱」並載入報表中。
7. **即時 CSV 追加存檔**：每完成一張圖片辨識即時寫入 CSV，且採用 `utf-8-sig` 編碼，**保證 Microsoft Excel 直接開啟中文不亂碼**。
8. **金鑰與模型診斷分頁**：整合 API 連線診斷，可一鍵查詢當前金鑰支援的所有可用模型清單，並清晰列出每個模型的**輸入與輸出 Token 限制上限 (Token Limits)** 與說明描述，幫助您精確掌握模型容量。
9. **依辨識結果重命名圖檔**：提供一鍵依據最新 CSV 結果自動重命名資料夾下圖檔之功能。採用精確的原檔名匹配、兩階段安全暫存命名（防止交換/覆蓋死鎖），且依預測物品名稱字母順序動態重新編號為 `1`、`2`、`3`...。

## 🛠️ 安裝說明

請確保您的電腦已安裝 **Python 3.9+**。

### 1. 複製或下載專案
將本專案目錄放置於您的本機目錄中（如 `C:\Users\charl\.gemini\antigravity\scratch\gemini-batch-tester`）。

### 2. 安裝依賴套件
* **圖形介面 (推薦)**：直接雙擊執行目錄中的 [install.bat](file:///C:/Users/charl/.gemini/antigravity/scratch/gemini-batch-tester/install.bat) 即可自動開始安裝。
* **命令列**：開啟終端機，進入專案目錄後執行：
  ```bash
  pip install -r requirements.txt
  ```

## 🚀 啟動與使用指南

### 1. 啟動程式
* **圖形介面 (推薦)**：直接雙擊執行目錄中的 [run.bat](file:///C:/Users/charl/.gemini/antigravity/scratch/gemini-batch-tester/run.bat) 即可一鍵開啟 GUI 應用程式。
* **命令列**：在專案目錄下執行：
  ```bash
  python main.py
  ```

### 2. 操作步驟
1. **輸入 API Key**：貼入您的 Google Gemini API Key。可點擊右側的 `👁` 切換隱藏與顯示。
2. **選取模型**：從下拉選單選擇您要測試的模型（預設為 `gemini-3.1-flash-lite-preview`），或直接於下拉框內輸入其他自訂的 Model ID。
3. **自訂 Prompt**：提示詞內可使用 `{model_id}` 變數，程式在發送請求時會自動將其替換為您當前選取的模型名稱。
4. **選取圖檔資料夾**：點擊「瀏覽...」按鈕選擇包含測試圖片的本機資料夾。
   * 💡 **小撇步**：若要進行對比分析，請將資料夾內的圖檔依 `物品名稱_分類_敘述_編號.jpg` 格式命名，例如：
     * `可樂_食品_氣泡_01.jpg` （原敘述為 "氣泡"，無過期標籤，比對輸出 `NA`）
     * `牛奶_食品_EX20260811_02.png` （原敘述為 "EX20260811"，將模型結果與 2026-08-11 比對，輸出 `OK`/`FAIL`）
     * `筆記本_文具_全新_03.webp` （原敘述為 "全新"，無過期標籤，比對輸出 `NA`）
5. **點擊「開始批次辨識」**：
   * 程式會在背景啟動測試，右側日誌區會實時滾動顯示模型回傳的原始 JSON 與解析狀態。
   * 進度條與狀態列會即時反映當前辨識速度。
   * 若需中途放棄，可隨時點擊「停止批次執行」按鈕安全中斷。
6. **檢視結果**：
   * 辨識完成後，系統會彈出成功提示。
   * 在您的圖檔資料夾下會產生一個名為 **`[實驗名稱]_[當前日期].csv`** 的檔案（例如 `20260531_Toys_20260531.csv`），使用 Microsoft Excel 或 任何 編輯器開啟即可檢視完美的對照報表！
7. **一鍵重新命名圖檔 (依結果更名)**：
   * 點擊「依辨識結果重命名圖檔」按鈕。
   * 程式將自動載入該資料夾中最新的結果 CSV 報表。
   * 依據 CSV 中原檔名進行精確匹配，並過濾掉辨識失敗或未知的項目。
   * 將成功的圖片重新命名為：`[預測名稱]_[預測類別]_[動態重新編號].原附檔名`，並動態將相同預測名稱的圖片依字母順序編號（`1`、`2`、`3`...）。
   * 過程中會以兩階段進行（Staging），先使用隨機暫時檔名防衝突，最後正式套用新名稱，保證不發生檔案互卡或覆蓋。

## 📊 CSV 欄位對照結構

產出的 CSV 包含以下 15 個欄位，依序排列（比對診斷列「名稱比對」、「類別比對」與「過期日期比對」位於最後三欄，方便直接觀看結果）：

1. **原圖檔檔名**：辨識圖檔的本機檔案名稱。
2. **原物品名稱**：由圖檔檔名中解析出的**真實名稱 (Ground Truth)**。
3. **原分類**：由圖檔檔名中解析出的**真實分類 (Ground Truth)**。
4. **原敘述**：由圖檔檔名中解析出的**真實敘述 (Ground Truth)**。
5. **原編號**：由圖檔檔名中解析出的**真實編號 (Ground Truth)**。
6. **模型 ID**：所使用的 Gemini 模型。
7. **辨識物品名稱**：由 Gemini API 預測之物品名稱。
8. **辨識分類**：由 Gemini API 預測之分類。
9. **辨識信心度**：由 Gemini API 預測之信心度。
10. **辨識有效日期**：由 Gemini API 預測之有效日期。
11. **辨識狀態**：`成功` 或 `失敗`。
12. **錯誤訊息**：若該圖檔呼叫出錯，記錄詳細例外原因。
13. **名稱比對**：名稱比對結果（`OK` 或 `FAIL`）。原物品名稱只要在**辨識物品名稱字串任一位置出現**，即判定為 `OK`。
14. **類別比對**：類別比對結果（`OK` 或 `FAIL`）。辨識分類必須與原分類**全文完全相同**才判定為 `OK`。
15. **過期日期比對**：有效日期比對結果（`OK`、`FAIL` 或 `NA`）。若原敘述以 `EX` 開頭，系統將自動進行日期字串標準化比對；若非 `EX` 開頭則輸出 `NA`。
